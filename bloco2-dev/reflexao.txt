# ATIVIDADE 7 — Make

# 1. O monitor histórico verificava a conclusão de cada job antes
#    de despachar o próximo. No make, qual mecanismo cumpre esse
#    papel? Como o make sabe que não precisa recompilar um arquivo
#    que não mudou?

Resposta 1:
No make, o mecanismo que cumpre o papel do monitor histórico é a verificação de **timestamps** e **dependências**. 
Para cada alvo, o make compara a data de modificação do arquivo alvo com a de seus arquivos dependentes. 
Se alguma dependência foi modificada mais recentemente que o alvo, o make executa os comandos para recompilar. 
Caso contrário, ele conclui que o arquivo está atualizado e não precisa recompilar, evitando trabalho desnecessário.


# 2. O monitor unificava "supervisor, carregador e rotinas de I/O
#    sob uma interface textual". O que o Makefile unifica no fluxo
#    de desenvolvimento moderno?

Resposta 2:
O Makefile unifica todas as etapas do fluxo de desenvolvimento em um único ponto de controle. 
Ele coordena a **compilação do código**, **limpeza de arquivos temporários**, **geração de relatórios** 
e outras tarefas automatizadas. Dessa forma, o desenvolvedor pode executar um único comando (`make`)
para reproduzir todo o build ou partes específicas do projeto, garantindo consistência e reprodutibilidade.
